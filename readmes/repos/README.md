# READMEs — Repos

Entry point documents for code repositories. One per repo. Written for developers — humans navigating a codebase for the first time or returning after time away.

## Audience

Developers. These documents assume technical fluency. They answer: what is this, how do I run it, and how do I contribute?

## Document Types

| Document | Purpose | Status |
|---|---|---|
| Repo README (`rdm-repo`) | The entry point for a codebase — what it does, how to set it up, how to run it, and where to go for contribution guidelines. Comes in several flavors depending on what kind of repo it is. | ✅ `rdm-repo_cli-pbj.md`, `rdm-repo_api-pbj.md`, `rdm-repo_mcp-pbj.md`, `rdm-repo_web-pbj.md`, `rdm-repo_mobile-pbj.md` |
| Contributing (`ctb`) | Architecture overview, directory structure, local dev setup, and testing guidelines for contributors. Lives at `CONTRIBUTING.md` in the repo root. | ✅ `ctb_cli-pbj.md` |
