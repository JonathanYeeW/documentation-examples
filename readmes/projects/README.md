# READMEs — Projects

Orientation documents for LLM project sessions. Written for models, not humans — these are the documents an LLM reads at the start of a session to understand a project's current state and pick up where the last session left off.

## Audience

LLMs. A human authors these, but the reader is a model. Written for cold-start clarity — the model has no session memory and no prior context. Everything it needs to be useful must be in this document.

## Structure

Project READMEs follow a fixed four-section structure, plus one project-specific section:

1. **Summary** — what the project is and what it's for
2. **What's Here** — the directory structure and what lives where
3. **Current State** — what's active, what's done, what's next
4. **To Pick Up the Work** — where to start, what to read first, any active blockers

The fifth section is named explicitly for what it contains — a post index, a list of people, an active ticket — whatever is most useful for this specific project type.
