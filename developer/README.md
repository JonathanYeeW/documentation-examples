# Developer

Setup guides and practical reference docs for working in a codebase. Not architecture or product context — the how-to documentation a developer reaches for when configuring their environment or performing a specific task for the first time.

## Audience

Developers, including new contributors getting set up. These documents assume basic technical literacy but not familiarity with this specific repo or toolchain. They answer: how do I get this working on my machine?

## Document Types

| Document | Purpose | Status |
|---|---|---|
| Knowledge Base (`kb`) | How to perform a specific setup task or configure a specific tool | ✅ `kb_vscode-shell-command.md` |
